GIT

What is git?

Git is a version control system. A version control sysytem is a tool used to track and manage changes over time. A (VCS) is divided into two:

  • Centralized version control system
  • Decentralized version control system

Git is useful to its users in different ways. Its used to:

  • Track changes
  • Collaborate with others on projects
  • Revert back to old versions or forms

Installing git

Let's get practical!
First, check whether git is already installed in your machine. To do this, open your terminal or command prompt from the start menu or from searching manually. Then, run git --version. This will tell you the version of git you are using. if it doesn't, use this link to download git in your machine. Afterwards run the command.

Configuring git

One down, many to go!
At this point, i am assuming you already have git installed. Now lets have it configured. Let us start with the username. To configure the username, run git config --global user.name "your preferred username"
To configure your email address, run git config --global user.email "your email address"
Confirming whether these two have been configured is very easy. Just run:
For the username git config user.name
For the email git config user.email

Initializing a git repository

Now that we have git installed and configured with our email and our username, lets initialize a git repository. Think of a git repository as a folder that we track all of our changes. There are two types of a git repo, a local repository and a remote repository. A local repository is the one that exists in your computer while the remote one is the one online in sites like github.

Lets initialize.
Go to the folder you want to have your project and run the terminal.
To initialize a git repo, run git status first to confirm there is no existing repository. You should only have one git repository for your project.
Now run git init to initialize a git repository.

You can confirm by again running git status

Adding files to the staging area

You are doing amazing! now we need to add files to the staging area. Think of the staging area like a waiting room, your code stays there waiting to be admitted. To do this, we use git add filename eg git add text.txt

From the screenshot above, you can see that when we added the .txt file it was ready for commit and it was in green, that is a tracked file meaning that it is now being monitored by git while the .html file was red until we added it to the staging area.When it was red it was an untracked file, meaning git is not monitoring them. You can also use git add . to shorten the whole process and make it easier.

Commiting changes

We are almost done here. Commiting is like 'saving' the changes you added earlier. To commit you need to use git commit -m "commit message". For the 'commit messages' should answer, 'What will this commit do?'. For example git commit -m "Add our files to..."

It is also possible to view all of our past commits. We use git log

And just like that we are done with that part!

Branches

In a nutshell, branching helps in creating a copy of a file without messing the original file. Branching can be used in many instances eg:

  1. During Collaboration
  2. When fixing errors
  3. When adding any possible features
  4. Testing new ideas

Creating a branch

Let us now see how to create these branches for better understanding. To create a branch, we run git branch branch_name

Viewing all branches

To view the existing branches in a repository, run git branch The starred and green one is the active branch

Switching active branches

Now that we have created our branch and made sure it is there, lets move from the main one to out newly made branch. To do this, we run git switch

Merging branches

Now that we have made changes to the file, we can now merge it with the original copy. To do this, we run git branch branch_name Remember, when merging you need to be in the branch you want to bring changes into. Then you use the name of the branch of which you want to bring changes from.

Deleting a branch

Now that we merged our branches, we can finally delete our created branch without affecting the original copy. to do this we run the command git branch -d branch_name
